Rico Basic Accordion

Overview

Este ejemplo ilustra como utilizar el funcionamiento Rico.Accordion para transformar un conjunto de divs en un primer componente de clase acordeon.

El Rico.Accordion es en realidad un componente muy sencillo construido fuera de los comportamientos Rico y efectos. Anhade los controladores de eventos necesarios en los respectivos divs para manejar los aspectos visuales de expansion, colapso y se cierne.
HTML Code

La estructura es un ejemplo de HTML div externa que contiene todos los paneles. Entonces, cada panel es de solo un par de DIVs (uno para la cabecera y otro para el contenido) envuelto en un DIV exterior. Usted puede utilizar distintos elementos divs. 
 
							<div id="accordionDiv">
								<div id="overviewPanel">
									<div id="overviewHeader">
										Overview
									</div>
									<div id="panel1Content">
										... content text ...
									</div>
								</div>
							</div>
Rico Code

Para fijar el componente Acordeon al div contenedor acordeon, construir un objeto Rico.Accordion y aprobar los titulos de los paneles y los contenidos de la misma. Esto es un poco diferente a la version anterior, pero permite mucha mas flexibilidad. Con el nuevo prototipo de clase Selector sigue siendo muy facil. 
							new Rico.Accordion( $$('div.panelheader'), $$('div.panelContent') );
								-or-

								new Rico.Accordion( $$('div.panelheader'), $$('div.panelContent'), 
													{panelHeight:200, 
														hoverClass: 'mdHover',
														selectedClass: 'mdSelected'}} );
El segundo ejemplo especifica la altura de los paneles de acordeon y las clases CSS que puede estar asociada con los comportamientos de acordeon. Hay muchos otros parametros de configuracion que se pueden especificar para modificar diversas aspectos visuales del acordeon. El panelHeight es el atributo que mas comunmente se invalida.
Important Note

El acordeon es muy flexible y puede manejar ahora barras de desplazamiento en firefox mediante el uso de algunas caracteristicas nuevas en Rico. Sin embargo, usted tiene que asegurarse de que los elementos de la cabecera y el contenido que pasa al constructor.Accordion tengan los mismos elementos y coincidan en el orden en que se pasan. El Acordeon nuevo requiere mucho menos atributos HTML que las versiones anteriores Rico.
